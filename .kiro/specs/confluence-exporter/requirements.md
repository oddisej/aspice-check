# Requirements Document — Confluence Page Exporter

## Introduction

This spec defines a **Python CLI tool** that exports Confluence pages to Markdown files with full support for embedded images, Gliffy diagrams, and AI-generated textual descriptions of visual content. The exported Markdown is designed to be consumed by the `aspice-eval` tool for ASPICE compliance gap analysis, where an AI agent needs to understand both textual and visual content (process flows, swimlanes, decision points) in SDP documentation.

### Motivation

- SDP and process documentation lives in Confluence, but the `aspice-eval` tool requires Markdown input
- Currently, exporting pages and separately exporting Gliffy diagrams as PNGs, then manually linking them into Markdown, is tedious and error-prone
- Gliffy diagrams contain critical process flow information (swimlanes, decision points, activity sequences) that an AI agent cannot interpret from images alone — textual descriptions are essential
- A single CLI command should produce a self-contained Markdown file with all images downloaded and described, ready for `aspice-eval` ingestion

### Key Integration Points

- **Upstream:** Confluence Cloud REST API
- **Downstream:** `aspice-eval` SDP ingester (`SDPIngester.ingest()`) which expects Markdown files with section headers
- **AI providers:** Multimodal AI models (Claude, GPT-4o) for generating textual descriptions of images and diagrams

## Glossary

- **Exporter**: The Python CLI tool that connects to Confluence, retrieves page content, converts it to Markdown, and downloads associated assets
- **Storage_Format**: Confluence's internal XHTML representation of page content, retrieved via the REST API `body.storage` expansion
- **Confluence_API**: The Confluence REST API used to retrieve pages, attachments, and metadata; accessed via the `atlassian-python-api` library
- **API_Token**: A Confluence Cloud API token used for authentication
- **Gliffy_Diagram**: A diagram created with the Gliffy plugin in Confluence, stored as a macro in Storage_Format with a PNG preview attachment
- **Attachment**: A file attached to a Confluence page, including images, Gliffy PNG previews, and other binary assets
- **Image_Description**: A textual description of an image or diagram generated by a multimodal AI model, providing semantic understanding of visual content for downstream AI processing
- **Output_Directory**: The local filesystem directory where the Exporter writes the Markdown file and downloaded image assets
- **Multimodal_Provider**: An AI model provider capable of accepting image input and producing textual descriptions (e.g., Anthropic Claude, OpenAI GPT-4o)

## Requirements

### Requirement 1: Confluence Authentication

**User Story:** As a user, I want to authenticate with my Confluence Cloud instance using an API token, so that the Exporter can access pages I have permission to view.

#### Acceptance Criteria

1. WHEN a Confluence Cloud base URL, email address, and API_Token are provided, THE Exporter SHALL authenticate with the Confluence_API using Basic Auth (email:token) and establish a valid session.
2. IF authentication fails due to invalid credentials, THEN THE Exporter SHALL return a descriptive error message identifying the authentication failure reason.
3. IF the Confluence base URL is unreachable, THEN THE Exporter SHALL return a descriptive error message indicating the connection failure.
4. THE Exporter SHALL accept authentication credentials via CLI arguments, environment variables, or a configuration file, with environment variables taking precedence over configuration file values.

### Requirement 2: Page Content Retrieval

**User Story:** As a user, I want to retrieve a Confluence page by its URL, so that the Exporter can access the page content for conversion.

#### Acceptance Criteria

1. WHEN a full Confluence Cloud page URL is provided, THE Exporter SHALL extract the page ID from the URL and retrieve the page content in Storage_Format via the Confluence_API endpoint `GET /rest/api/content/{id}?expand=body.storage,metadata.labels,version`.
2. WHEN a page is retrieved, THE Exporter SHALL also retrieve the list of Attachments associated with the page via the Confluence_API.
3. IF the provided URL does not match a valid Confluence Cloud page URL pattern, THEN THE Exporter SHALL return a descriptive error message identifying the expected URL format.
4. IF the specified page does not exist or the authenticated user lacks read permission, THEN THE Exporter SHALL return a descriptive error message identifying the access issue.

### Requirement 3: Storage Format to Markdown Conversion

**User Story:** As a user, I want Confluence page content converted to clean Markdown, so that the output is readable and compatible with the `aspice-eval` ingester.

#### Acceptance Criteria

1. WHEN a page in Storage_Format is provided, THE Exporter SHALL convert the XHTML content to Markdown format.
2. THE Exporter SHALL convert the following Storage_Format elements to their Markdown equivalents: headings (h1–h6), paragraphs, bold and italic text, ordered and unordered lists, tables, code blocks, hyperlinks, and horizontal rules.
3. THE Exporter SHALL preserve the hierarchical heading structure from the Confluence page so that the `aspice-eval` SDP ingester can extract section headers.
4. WHEN a Storage_Format element has no Markdown equivalent, THE Exporter SHALL render the element as plain text with a comment indicating the original element type.
5. THE Exporter SHALL produce a Markdown pretty-printer output that, when parsed back, preserves the semantic structure of the original content (round-trip structural equivalence).

### Requirement 4: Embedded Image Handling

**User Story:** As a user, I want all images from the Confluence page downloaded locally, so that the exported Markdown is self-contained.

#### Acceptance Criteria

1. WHEN the Storage_Format contains inline images referencing Attachments, THE Exporter SHALL download each image file to the Output_Directory.
2. WHEN the Storage_Format contains inline images referencing external URLs, THE Exporter SHALL download each external image to the Output_Directory.
3. THE Exporter SHALL replace image references in the Markdown output with relative paths pointing to the downloaded image files in the Output_Directory.
4. THE Exporter SHALL organize downloaded images in a subdirectory named `images/` within the Output_Directory.
5. IF an image download fails, THEN THE Exporter SHALL log a warning, include a placeholder in the Markdown output indicating the failed download, and continue processing the remaining content.
6. THE Exporter SHALL preserve the original file extension of each downloaded image.

### Requirement 5: Gliffy Diagram Export

**User Story:** As a user, I want Gliffy diagrams exported as PNG images, so that process flow diagrams are included in the Markdown output.

#### Acceptance Criteria

1. WHEN the Storage_Format contains a Gliffy macro, THE Exporter SHALL identify the corresponding PNG preview Attachment for the diagram.
2. WHEN a Gliffy PNG preview Attachment is identified, THE Exporter SHALL download the PNG file to the `images/` subdirectory in the Output_Directory.
3. THE Exporter SHALL replace the Gliffy macro in the Markdown output with a Markdown image reference pointing to the downloaded PNG file.
4. WHEN a Gliffy diagram has a title or name attribute in the macro, THE Exporter SHALL use that name as the image alt-text and filename base.
5. IF a Gliffy diagram has no PNG preview Attachment available, THEN THE Exporter SHALL log a warning and include a placeholder in the Markdown output indicating the missing diagram.

### Requirement 6: AI-Generated Image Descriptions

**User Story:** As a user, I want each exported image to include a textual description generated by an AI model, so that the `aspice-eval` AI agent can understand visual content like process flows, swimlanes, and decision points.

#### Acceptance Criteria

1. WHEN an image is downloaded, THE Exporter SHALL send the image to the configured Multimodal_Provider to generate an Image_Description.
2. THE Image_Description SHALL describe the semantic content of the image, including: diagram type (flowchart, swimlane, sequence diagram, etc.), key elements and their relationships, decision points and branches, and any visible text labels.
3. THE Exporter SHALL embed the Image_Description in the Markdown output directly below the image reference, formatted as a blockquote prefixed with "**Image Description:**".
4. WHEN generating an Image_Description for a Gliffy diagram, THE Exporter SHALL include a prompt instructing the Multimodal_Provider to focus on process flow elements: activities, decision points, swimlanes/roles, inputs, outputs, and transitions.
5. IF the Multimodal_Provider is unavailable or returns an error, THEN THE Exporter SHALL log a warning, include a placeholder description stating "Image description unavailable", and continue processing.
6. WHERE the user disables AI description generation via a CLI flag, THE Exporter SHALL skip Image_Description generation and include only the image reference with alt-text.

### Requirement 7: CLI Interface

**User Story:** As a user, I want a command-line interface to run the exporter, so that I can integrate it into scripts and automation workflows.

#### Acceptance Criteria

1. THE Exporter SHALL provide a CLI command `confluence-export` that accepts the following required arguments: Confluence page URL and Output_Directory path.
2. THE Exporter SHALL accept optional CLI arguments for: authentication credentials (email, API_Token), Multimodal_Provider selection (provider name, model, API key), and a flag to disable AI Image_Description generation.
3. WHEN the CLI is invoked with valid arguments, THE Exporter SHALL export the specified page and write the Markdown file and images to the Output_Directory.
4. WHEN the export completes successfully, THE Exporter SHALL print a summary to stdout listing: the output Markdown file path, the number of images downloaded, the number of Image_Descriptions generated, and any warnings encountered.
5. IF required arguments are missing, THEN THE Exporter SHALL display a usage message describing all available arguments and exit with a non-zero exit code.
6. THE Exporter SHALL return exit code 0 on success and a non-zero exit code on failure.

### Requirement 8: Output File Structure

**User Story:** As a user, I want the exported files organized in a predictable structure, so that I can reliably pass them to the `aspice-eval` tool.

#### Acceptance Criteria

1. THE Exporter SHALL write the converted Markdown to a file named `{page_title}.md` in the Output_Directory, where `{page_title}` is the sanitized Confluence page title (spaces replaced with underscores, special characters removed).
2. THE Exporter SHALL write all downloaded images to an `images/` subdirectory within the Output_Directory.
3. THE Exporter SHALL name downloaded image files using the pattern `{sanitized_original_name}.{extension}`, preserving the original filename where possible and resolving name collisions by appending a numeric suffix.
4. WHEN the Output_Directory does not exist, THE Exporter SHALL create it along with the `images/` subdirectory.
5. THE Exporter SHALL include a YAML front-matter block at the top of the Markdown file containing metadata: source Confluence URL, page ID, page title, export timestamp, and exporter version.

### Requirement 9: Error Handling and Logging

**User Story:** As a user, I want clear error messages and logging, so that I can diagnose issues when an export fails.

#### Acceptance Criteria

1. THE Exporter SHALL log all operations at the INFO level, including: page retrieval, image downloads, Gliffy diagram detection, and Image_Description generation.
2. WHEN a non-fatal error occurs (single image download failure, single Image_Description generation failure), THE Exporter SHALL log the error at WARNING level and continue processing.
3. IF a fatal error occurs (authentication failure, page not found, network unreachable), THEN THE Exporter SHALL log the error at ERROR level and exit with a non-zero exit code.
4. THE Exporter SHALL support a `--verbose` flag that increases log output to DEBUG level, including API request and response details.
5. THE Exporter SHALL write log output to stderr so that stdout remains clean for structured output and piping.
